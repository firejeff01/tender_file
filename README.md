# 標案文件產生器

依 Excel 中的標案資料，自動產生一組制式 ODT 文件（政府要求的 ODF 格式），
並把文件內的公司固定資料與每案變動資料（機關名稱、案件名稱、簽署日期等）
一次填好。設計目標是讓行政人員在**不需安裝任何軟體**的環境下也能使用。

## 特色

- **免安裝、綠色可攜**：目標 .NET Framework 4.8（Windows 10 1903+／Windows 11
  作業系統內建元件），整個資料夾複製即可用，雙擊執行，不需站台、不需安裝檔、
  不需系統管理員權限。
- **零 Office 依賴**：執行期以純 zip + XML 直接讀寫 `.xlsx` 與 `.odt`，
  不透過 Word / Excel / LibreOffice 轉檔，格式 100% 保真。
- **可用系統內建編譯器重建**：附 `建置.bat`，用 Windows 內建的 `csc.exe`
  即可從原始碼重新編譯（不需 Visual Studio、不需連外網），方便公司 IT
  在白名單環境自行稽核與重建。
- **符合 ODF 規範**：ODT 重打包時 `mimetype` 為第一個 entry 且 stored，
  可通過國發會 ODF 文件格式檢測。
- **可自助新增範本**：程式內建「新增範本」，選一份把欄位標了顏色的 ODT，
  即自動正規化成 `${參數}` 範本並更新 Excel（加欄/加列），不需工程師介入。

## 使用（行政人員）

1. 開啟 `標案資料.xlsx`，於「公司資料」工作表填入公司固定資料（只需一次）。
2. 於「標案清單」工作表新增一列標案，存檔並關閉 Excel。
3. 雙擊 `標案文件產生器.exe`，勾選要產生的標案，按「產生文件」。
4. 文件會輸出到「輸出資料夾」（預設為程式旁的 `輸出\`，可在程式中變更）。

詳見 [`使用說明.txt`](使用說明.txt)。

## 建置（資訊人員）

雙擊 `建置.bat` 即可，它會呼叫系統內建的：

```
%WINDIR%\Microsoft.NET\Framework64\v4.0.30319\csc.exe
```

編譯 `原始碼\*.cs` 產生 `標案文件產生器.exe`。原始碼為 C# 5 語法以相容內建編譯器。
技術細節見 [`資訊人員說明.txt`](資訊人員說明.txt)。

自我測試：`標案文件產生器.exe /selftest`（產生 `selftest.log`，exit code 為失敗數）。

## 專案結構

```
標案文件產生器.exe   免安裝執行檔（可用 建置.bat 重建）
標案資料.範例.xlsx   Excel 格式範本（版控保留；資料為通用佔位符）
標案資料.xlsx        實際填寫的工作檔（已被 .gitignore 排除，不進版控）
範本\                已正規化的 ODT 範本（含 ${參數} 佔位符）＋ tokens.txt
原始碼\              C# 原始碼、app.manifest、建置設定
工具\                normalize.ps1（範本正規化）、make_xlsx.ps1（產生初始 Excel）
使用說明.txt         行政人員操作說明
資訊人員說明.txt      IT 技術與建置說明
```
